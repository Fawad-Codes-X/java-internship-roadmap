# Java Basics: JDK, JRE, JVM, javac, java, and Bytecode

## 1. JDK (Java Development Kit)

The **JDK** is a complete toolkit used to develop Java applications.

It contains:

* JRE (Java Runtime Environment)
* JVM (Java Virtual Machine)
* `javac` compiler
* Other development tools such as debugger and documentation generator

**Use the JDK when you want to write, compile, and run Java programs.**

---

## 2. JRE (Java Runtime Environment)

The **JRE** provides everything needed to run Java applications.

It contains:

* JVM
* Java libraries
* Supporting files

The JRE **cannot compile** Java source code. It is only used to execute Java programs.

---

## 3. JVM (Java Virtual Machine)

The **JVM** is the engine that executes Java programs.

Its responsibilities include:

* Loading class files
* Verifying bytecode
* Managing memory
* Running garbage collection
* Converting bytecode into machine code using the Interpreter and JIT Compiler

Because every operating system has its own JVM implementation, the same Java program can run on Windows, Linux, or macOS without changing the source code.

---

## 4. javac (Java Compiler)

`javac` is the Java compiler included in the JDK.

It converts Java source code (`.java`) into bytecode (`.class`).

Example:

```bash
javac Hello.java
```

Output:

```
Hello.class
```

---

## 5. java Command

The `java` command starts the JVM and runs a compiled Java program.

Example:

```bash
java Hello
```

The JVM loads the `Hello.class` file, verifies it, executes the bytecode, and displays the program output.

---

## 6. Bytecode

**Bytecode** is the intermediate code generated by the Java compiler.

* Source file: `Hello.java`
* Compiled file: `Hello.class`

Bytecode is **platform-independent**, which means it can run on any operating system that has a compatible JVM.

This is the reason Java follows the principle:

**Write Once, Run Anywhere (WORA).**

---

## Java Execution Flow

```
Write Java Code (.java)
        │
        ▼
Compile using javac
        │
        ▼
Bytecode (.class)
        │
        ▼
Run using java command
        │
        ▼
JVM Starts
        │
        ▼
Class Loader
        │
        ▼
Bytecode Verifier
        │
        ▼
Execution Engine
(Interpreter + JIT Compiler)
        │
        ▼
Machine Code
        │
        ▼
CPU Executes Program
```

## Summary

* **JDK** → Used to develop Java programs (includes JRE, JVM, and compiler).
* **JRE** → Used to run Java programs.
* **JVM** → Executes bytecode and manages program execution.
* **javac** → Compiles `.java` files into `.class` bytecode.
* **java** → Launches the JVM and runs compiled Java programs.
* **Bytecode** → Platform-independent code that enables Java's "Write Once, Run Anywhere" capability.
